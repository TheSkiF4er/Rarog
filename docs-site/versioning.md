# Versioning & Support

Rarog 3.x использует семантическое версионирование (SemVer) и фиксирует
гарантии поддержки ветки.

## Семантическое версионирование

Формат: `MAJOR.MINOR.PATCH`

- `MAJOR` (например, 2 → 3):
  - может включать breaking changes;
  - требует миграционного гайда.
- `MINOR` (например, 2.0 → 2.1):
  - добавляются новые компоненты, утилиты, токены, улучшения;
  - **не** ломает уже задокументированный API.
- `PATCH` (например, 2.0.0 → 2.0.1):
  - только багфиксы и внутренние улучшения без изменения публичного API.

Выпуск **3.0.0** фиксирует «зрелое» состояние фреймворка для ветки 3.x. Всё, что описано в
разделе API Reference, считается стабильным и поддерживаемым в рамках ветки 3.x.

## Политика поддержки ветки 3.x

- Безопасность (security):
  - критические уязвимости закрываются в актуальной минорной версии 2.x.
- Багфиксы:
  - исправляются в patch-релизах 2.x, не меняющих API.
- Совместимость:
  - новых breaking changes не будет без перехода на 3.x и отдельного миграционного гайда.

## Tailwind + Bootstrap альтернатива

Rarog позиционируется как гибридная альтернатива связке **Tailwind + Bootstrap**:

- **Design tokens**:
  - централизованный слой цветов, spacing, тени, брейкпоинтов, экспортируемый в `rarog.tokens.json`.
- **Utilities**:
  - utility-first подход в духе Tailwind (responsive/state-префиксы, JIT, arbitrary values).
- **Components**:
  - готовые компоненты уровня Bootstrap (buttons, alerts, nav, forms, grid и др.).
- **JS Core**:
  - ванильное JS-ядро без jQuery (dropdown, collapse, modal и т.д.).

При этом Rarog хорошо интегрируется:

- с Cajeer-экосистемой (проекты Cajeer Team),
- с любыми сторонними стекками: Laravel, React, Vite, статические сайты и др.

Цель 2.x — быть стабильной основой для дизайн-систем и продуктовых UI, а не только «игрушкой для экспериментов».


## Версионирование документации

- `/v3/...` — основная, актуальная ветка документации для Rarog 3.x;
- `/v2/...` — справочные материалы по 2.x (legacy), которые могут быть
  доступны как отдельный билд docs или ветка репозитория.

Рекомендуется:

- для новых проектов ссылаться на Rarog 3.x и /v3 docs;
- для старых проектов, сидящих на 2.x, использовать migration-гайд и
  постепенно переходить на 3.x.

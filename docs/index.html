Theming & Design Tokens

Rarog строится вокруг CSS-переменных (дизайн-токенов) и поддерживает несколько тем "из коробки": default, dark, contrast.

Базовые токены и шкалы

Базовые цветовые шкалы определены в packages/core/src/tokens/_color.css: --rarog-color-primary-50 … -900 и аналогичные шкалы для secondary, success, danger, warning, info.

Поверх шкал определены semantic alias'ы: --rarog-color-bg-elevated, --rarog-color-border-subtle, --rarog-color-focus-ring и др. Компоненты и утилиты ориентируются именно на semantic-токены, что упрощает создание новых тем.

Как включить тему

В сборке доступны минимум три темы:

  • rarog-theme-default.css — светлая тема по умолчанию (селектор :root).
  • rarog-theme-dark.css — тёмная тема (селектор :root.theme-dark).
  • rarog-theme-contrast.css — high-contrast тема (селектор :root.theme-contrast).

Для переключения темы достаточно добавить класс на корневой элемент: 

<html class="theme-dark">
  ...
</html>

<html class="theme-contrast">
  ...
</html>
Своя тема на базе Rarog

Есть два основных пути создать свою тему:

  1. Переопределить переменные в своём CSS.
    Подключите базовые токены и утилиты, а затем в отдельном файле переопределите нужные переменные, например: 
    :root.my-brand {
  --rarog-color-primary: #8b5cf6;
  --rarog-color-bg: #0b1020;
  --rarog-color-text: #f9fafb;
}
    После этого добавьте класс my-brand на <html> или <body>.
  2. Создать отдельный файл темы.
    Скопируйте один из файлов тем (например rarog-theme-dark.css) в packages/themes/src/rarog-theme-mybrand.css, адаптируйте значения переменных и добавьте новую тему в конфигурацию сборки rarog.config.json.
JSON-токены для design tools

Файл rarog.tokens.json в корне репозитория содержит те же токены в JSON-формате (цветовые шкалы, spacing, радиусы и тени). Его можно использовать как источник правды для Figma-плагинов или любых инструментов работы с Design Tokens.

Config & CLI

Начиная с версии 1.6.0, Rarog поддерживает Tailwind-подобный конфиг rarog.config.js/ts и CLI-утилиту rarog, которая умеет собирать токены и запускать документацию.

rarog.config.js / rarog.config.ts

Конфиг хранит дизайн-токены и брейкпоинты: 

// rarog.config.js
/** @type {import("./rarog.config.types").RarogConfig} */
const config = {
  theme: {
    colors: {
      primary: { /* шкала 50–900 */ },
      secondary: { /* ... */ },
      // success / danger / warning / info / semantic
    },
    spacing: {
      0: "0",
      1: "0.25rem",
      2: "0.5rem",
      // ...
    },
    radius: {
      xs: "0.125rem",
      sm: "0.25rem",
      md: "0.5rem",
      lg: "0.75rem",
      xl: "1rem",
      full: "9999px"
    },
    shadow: {
      xs: "0 1px 2px rgba(15, 23, 42, 0.08)",
      sm: "0 2px 4px rgba(15, 23, 42, 0.12)",
      md: "0 4px 6px rgba(15, 23, 42, 0.16)",
      lg: "0 10px 15px rgba(15, 23, 42, 0.25)"
    }
  },
  screens: {
    sm: "640px",
    md: "768px",
    lg: "1024px",
    xl: "1280px",
    "2xl": "1536px"
  },
  extend: {
    colors: {},
    spacing: {},
    radius: {},
    shadow: {}
  },
  plugins: []
};

module.exports = config;
Rarog CLI

CLI добавляется через поле bin в package.json и предоставляет одну команду rarog с подкомандами: 

npx rarog build   # сгенерировать CSS-токены и rarog.tokens.json из rarog.config.js
npx rarog init    # создать rarog.config.js/ts и пример проекта (examples/starter)
npx rarog docs    # запустить dev-документацию (tools/docs-dev.mjs)
Связка config → tokens

Команда rarog build читает rarog.config.js, объединяет его с дефолтной темой и пересобирает:

  • packages/core/src/tokens/_color.css
  • packages/core/src/tokens/_spacing.css
  • packages/core/src/tokens/_radius.css
  • packages/core/src/tokens/_shadow.css
  • packages/core/src/tokens/_breakpoints.css
  • rarog.tokens.json — JSON-представление токенов для design tools.

Для изменения палитры, spacing или брейкпоинтов теперь достаточно поменять значения в rarog.config.js и выполнить rarog build — остальной код фреймворка продолжит использовать CSS-переменные.

Build modes

Начиная с версии 1.7.0, Rarog умеет собирать CSS в двух режимах: full (полная сборка) и jit (JIT / tree-shaking).

Режим full

В режиме full используется полный набор CSS-файлов: rarog-core.css, rarog-utilities.css, rarog-components.css, а команда rarog build отвечает только за пересборку дизайн-токенов (_color.css, _spacing.css, _radius.css, _shadow.css, _breakpoints.css, rarog.tokens.json).

Это режим по умолчанию, удобный на этапе разработки дизайн-системы и компонентов. 

// rarog.config.js
module.exports = {
  mode: "full",
  content: ["./resources/**/*.{html,php,js,jsx,ts,tsx,vue}"],
  theme: { /* ... */ },
  screens: { /* ... */ }
};
Режим jit (JIT / Tree-shaking)

В режиме jit CLI анализирует файлы проекта (из секции content в rarog.config.js), собирает список реально используемых классов и на основе него формирует минимальный CSS-файл dist/rarog.jit.css. 

// rarog.config.js
module.exports = {
  mode: "jit",
  content: ["./resources/**/*.{html,php,js,jsx,ts,tsx,vue}"],
  theme: { /* ... */ },
  screens: { /* ... */ }
};

// Сборка токенов + JIT-CSS
npx rarog build

// Результат:
//  - rarog.tokens.json + CSS-токены (как в режиме full)
//  - dist/rarog.jit.css — урезанный CSS под конкретный проект

В JIT-файле всегда присутствуют дизайн-токены (CSS-переменные) и только те утилиты/компоненты, классы которых были найдены в исходниках. Таким образом, для небольшого проекта размер CSS становится заметно меньше по сравнению с полной сборкой.

Arbitrary values

JIT-режим также поддерживает простой вариант произвольных значений по Tailwind-подобному синтаксису. Классы с форматами w-[...], h-[...], bg-[...], text-[...] автоматически превращаются в CSS-правила.

Примеры произвольных значений 
<div class="w-[320px] h-[200px] bg-[#0f172a] text-[white]">
  Бокс фиксированного размера с произвольным цветом фона и текста
</div>

При JIT-сборке будет сгенерирован CSS следующего вида: 

.w-\[320px\] { width: 320px; }
.h-\[200px\] { height: 200px; }
.bg-\[#0f172a\] { background-color: #0f172a; }
.text-\[white\] { color: white; }

На текущем этапе произвольные значения поддерживаются для размеров (w-[...], h-[...]) и цветов (bg-[...], text-[...]). Этого достаточно, чтобы описывать редкие, нестандартные случаи, не засоряя общий набор токенов.

JavaScript

Rarog JS Core v1.5.0 даёт базовую интерактивность для компонентов без jQuery. Один бандл rarog.js включает поддержку dropdown, collapse/accordion и modal.

Подключение

Самый простой вариант — подключить UMD-бандл как обычный скрипт: 

<script src="../packages/js/dist/rarog.js"></script>

Для модульного окружения (ESM) используйте: 

<script type="module">
  import Rarog, { Modal } from "../packages/js/dist/rarog.esm.js";

  const modalEl = document.getElementById("demo-modal");
  const modal = Modal.getOrCreate(modalEl);
  modal.show();
</script>
Data-API

Все интерактивные компоненты работают через data-атрибуты. Rarog автоматически инициализирует обработчики на DOMContentLoaded.

Dropdown

<div class="dropdown">
  <button
    class="btn btn-secondary dropdown-toggle"
    data-rg-toggle="dropdown"
    data-rg-target="#userMenu"
  >
    Menu
  </button>
  <div id="userMenu" class="dropdown-menu" hidden>
    <a class="dropdown-item" href="#">Profile</a>
    <a class="dropdown-item" href="#">Settings</a>
  </div>
</div>

Collapse / Accordion

<button
  class="btn btn-outline"
  data-rg-toggle="collapse"
  data-rg-target="#faqItem1"
>
  Показать ответ
</button>
<div id="faqItem1" class="collapse mt-2" hidden>
  Ответ на вопрос
</div>

Modal

<button
  class="btn btn-primary"
  data-rg-toggle="modal"
  data-rg-target="#demoModal"
>
  Открыть модалку
</button>

<div class="modal" id="demoModal" aria-hidden="true">
  <div class="modal-dialog">
    <div class="modal-header">
      <h2 class="card-header">Заголовок</h2>
    </div>
    <div class="modal-body">
      Тело модального окна.
    </div>
    <div class="modal-footer">
      <button class="btn btn-secondary" data-rg-dismiss="modal">Закрыть</button>
    </div>
  </div>
</div>
JS-API

Помимо Data-API, каждую сущность можно управлять программно через JS-API: 

const trigger = document.querySelector("[data-rg-toggle='dropdown']");
const dropdown = Rarog.Dropdown.getOrCreate(trigger);
dropdown.show();

const panelTrigger = document.querySelector("[data-rg-toggle='collapse']");
const collapse = Rarog.Collapse.getOrCreate(panelTrigger);
collapse.toggle();

const modalEl = document.getElementById("demoModal");
const modal = Rarog.Modal.getOrCreate(modalEl);
modal.show();
Доступность (a11y)

JS Core автоматически выставляет базовые ARIA-атрибуты:

  • Dropdown: aria-haspopup="menu", aria-expanded, role="menu".
  • Collapse: aria-expanded на триггере, role="region" и aria-hidden на панели.
  • Modal: role="dialog", aria-modal="true", aria-hidden, фокус-ловушка и закрытие по ESC.

Клавиатурная навигация:

  • Dropdown: стрелки вверх/вниз перемещают фокус по пунктам меню, ESC закрывает меню.
  • Modal: TAB циклически перемещает фокус внутри модального окна, ESC закрывает модалку.

# Contributor Guide

This section is for contributors only.

Contributor documentation should not be mixed with end-user documentation.

## What belongs here

- local development setup
- repository structure
- contribution workflow
- testing and validation
- release process
- docs workflow

## Suggested contributor topics

### Local development

Explain how to bootstrap the repo, install dependencies, and run local checks.

### Architecture

Describe package boundaries, build flow, runtime model, and key design decisions.

### Contribution workflow

Document branches, pull request expectations, code review norms, and release hygiene.

### Docs workflow

Explain how documentation pages are added, reviewed, and kept aligned with product changes.

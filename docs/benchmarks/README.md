# Программа бенчмарков

«Рарог» больше не должен лишь *казаться* быстрым. Этот раздел фиксирует единую программу бенчмарков и связывает её с бэклогом оптимизаций и выпуск gates.

## Метрики

- **Скорость сборки** — полный CSS/JS build path
- **Скорость JIT** — скорость локальной генерации проекта
- **Размер output** — итоговый размер пригодный для публикации CSS output
- **Стоимость рантайма** — цена среда выполнения orchestration
- **Стоимость переключения темы** — переключение под стороннюю марку темы
- **Накладные расходы рендера компонентов** — влияние Компонент render overhead

## Сравниваемые стеки

- Bootstrap
- Tailwind
- UnoCSS
- shadcn/ui
- Chakra UI
- MUI

## Правила

1. Любое утверждение об оптимизации должно ссылаться на категорию метрики.
2. Любая регрессия выше согласованного порога должна попадать в очередь задач.
3. Релиз без сравнительный замер snapshot для затронутого pipeline не считается завершённым.

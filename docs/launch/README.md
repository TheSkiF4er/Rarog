# Позиционирующий запуск Рарог 1.0

Рарог 1.0 должен запускаться не как «ещё один utility framework», а как **система оформления platform для продуктовых команд с несколькими темами, под стороннюю марку сценариями и dashboard/admin нагрузкой**.

## Чёткое позиционирование

**Рарог** — это платформа для команд, которым нужно одновременно:

- быстро собирать product UI без разрастания кастомного CSS;
- поддерживать несколько тем и под стороннюю марку брендов;
- выпускать dashboard/admin и SaaS Поверхностьs без бесконечной ручной сборки система оформления;
- держать production-качество: a11y, дисциплину бандлов, стабильные экспорты и понятный migration path.

## Почему сейчас

Большинство конкурентов хорошо решают только одну часть задачи:

- скорость с опорой на вспомогательные классы разработки;
- DX компонентов;
- темизация;
- потребности enterprise Панель-продуктов.

Рарог 1.0 объединяет это в один stack:

1. **вспомогательные классы + Компонентs + tokens + themes** в одной системе;
2. **конструктор тем / token editor** как порядок работы для заказчиков;
3. **расширение SDK** для расширения платформы;
4. **средство командной строки Pro tools** для migration, проверка и выпуск hygiene;
5. **сравнительный замер program + quality bar** как доказательство зрелости.

## Где Рарог выигрывает

### 1. Многотемный SaaS

Один кодбейс, несколько tenant themes, предсказуемое переключение тем и токены как source of truth.

### 2. Под стороннюю марку B2B приложения

Быстрый брендовый онбординг, import/export тем, review flows для procurement и enterprise delivery.

### 3. Внутренние дашборды и админки

Плотные интерфейсы, формы, таблицы, аналитические блоки, Панель kits и admin-focused starters.

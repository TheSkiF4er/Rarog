# Migration

This section helps existing users move from the old documentation structure and older APIs to the current model.

## Migration notes

- User docs and contributor docs are now split
- GitBook-style structure is now the canonical docs IA
- Legacy mixed-content pages should be retired after migration

## What to check during migration

Review:
- internal links
- old entry points
- duplicated pages
- contributor content mixed into user docs
